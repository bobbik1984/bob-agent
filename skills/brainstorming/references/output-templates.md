# 输出模板参考

## 模板 A：标准分析报告

```
# 分析报告：[问题标题]

## 问题定义
[一句话重新陈述核心问题]

**分析目标：** [决策型 / 优化型 / 探索型]
**选用视角：** [模块1] + [模块2] + ...

---

## [模块1] 分析

[结构化输出，见各模块执行指南]

---

## [模块2] 分析

[结构化输出]

---

## 综合评估

**核心矛盾：**
[各视角中反复出现的关键张力]

**关键权重因素：**
[哪些因素对当前目标影响最大，及理由]

---

## 决策建议

【推荐方向】
[一句话结论]

【核心理由】
1. [理由一]
2. [理由二]
3. [理由三（可选）]

【主要风险】
- [风险一及应对思路]
- [风险二及应对思路]

【下一步行动】
1. [具体行动一]
2. [具体行动二]
3. [具体行动三（可选）]
```

## 模板 B：方案对比表格

适用于有 2~4 个明确备选方案的决策场景。

```markdown
| 评估维度 | 方案 A | 方案 B | 方案 C |
|---------|--------|--------|--------|
| 成本     | 低 ✓   | 高 ✗   | 中     |
| 实施周期 | 3个月  | 6个月  | 4个月  |
| 风险等级 | 中     | 低 ✓   | 高 ✗   |
| 战略契合 | 高 ✓   | 中     | 中     |
| 可扩展性 | 低 ✗   | 高 ✓   | 高 ✓   |
| **综合** | ⭐⭐⭐  | ⭐⭐⭐⭐ | ⭐⭐⭐  |

**推荐：方案 B**  
理由：[一句话说明]
```

## 模板 C：快速头脑风暴（轻量版）

适用于问题相对简单、用户只需要快速理清思路的场景。

```
**问题：** [用户问题]

**正向因素（为什么可以做）：**
- [因素1]
- [因素2]
- [因素3]

**反向因素（为什么要谨慎）：**
- [因素1]
- [因素2]

**关键判断：**
[1~2句话，说明倾向哪个方向及核心理由]

**如果要行动，先做这件事：**
[一个最小可行的下一步]
```

---

## 评分规范

在对比分析中使用统一的评级体系：

| 符号 | 含义 |
|------|------|
| ✓ / 绿 | 明显优势或满足要求 |
| ✗ / 红 | 明显劣势或不满足要求 |
| ~ / 黄 | 中性或不确定 |
| ⭐ ~ ⭐⭐⭐⭐⭐ | 综合评分（1~5星） |
| 高/中/低 | 相对评级，无法量化时使用 |

---

## 使用建议

- 简单问题 → 模板 C（快速版）
- 单方案评估 → 模板 A（完整报告）
- 多方案比较 → 模板 A + 模板 B（嵌入对比表）
- 用户明确要求"给个报告" → 用模板 A，可直接导出为文档
