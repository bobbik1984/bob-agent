# 知识收割机 (AKP Link Harvester) v2.0 (Bob-Agent Native)

## 双模式工作流

### 模式 A：快速捕获（默认模式）

**触发条件**：用户发送了一个 URL 链接，且没有说"收录"、"保存"等完整处理指令。

**行为**：
**绝对不要询问用户！不要等待确认！**
立即使用 `write_file` 或 `append_file`（若有）将链接、捕捉时间及用户可能的短备注追加到你当前设定的 **工作目录(Workspace)** 下的 `inbox/pending_links.md` 文件中。如果未设定工作目录，请先提醒用户配置。
向用户简短汇报："📥 已极速存入收件箱！"

### 模式 B：完整收录（显式触发）

**触发条件**：用户的消息中包含"收录"、"保存"、"记下来"、"剪报"等明确指令及链接。

**行为**：立即执行完整处理流程（S.O.P），不进入 inbox 排队。

**S.O.P 执行步骤**：

**Step 1: 提取纯净文本**

调用 `tinyfish_fetch` 工具，传入用户提供的 `url`。该工具会自动绕过反爬虫，并返回极其纯净的 Markdown 正文和标题。

**Step 2: 生成摘要 (Agent 大脑内部思考)**

你已经在 Step 1 拿到了 Markdown 文本。现在，请发挥你的总结能力，将长文本提炼为不超过 300 字的核心摘要，并列出 3 个关键论点。

**Step 3: 大脑思考与精准打标**

根据你阅读的文本内容，自行挑选 1 个主分类标签和 2-3 个子标签（如 `Tech`, `AI`, `Investment` 等）。

**Step 4: 规范化落库**

调用 `write_file` 工具，将处理好的结果写成一篇标准化的 Markdown 文档。
- **保存路径**: 当前设定的 **工作目录(Workspace)** 下的 `raw/article/[生成一个基于标题或时间戳的合法英文或拼音文件名].md`。如果未设定工作目录，请提示用户。
- **内容格式要求**：
  1. 必须以严格的 YAML Frontmatter 开头。
  2. 正文部分的核心实体（人名、公司名、专有名词）必须使用 `[[双向链接]]` 语法包裹，以便构建 LLM Wiki 知识库图谱。

**模板参考**:
```markdown
---
date: "[使用 system_time 工具获取今天日期，格式 YYYY-MM-DD]"
source: "[原始链接]"
tags: [tag1, tag2, tag3]
---

# [文章标题]

## 💡 核心摘要
[Step 2 的 300 字摘要，注意用 [[实体]] 包裹核心词汇]

## 🔑 关键论点
- 论点 1 (包含 [[实体]])
- 论点 2
- 论点 3

## 📄 原始正文
[Step 1 返回的全部 Markdown 内容，保持原样，也可以为其中的重要词汇加上 [[双链]]]
```

执行完毕后，向用户简短汇报："✅ 归档成功！已存入 raw/article/，标签：[你选的标签]，并自动启用了 Wiki 双向链接格式。"。
