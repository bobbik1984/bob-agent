### ?? 核心疑问极速解答
 1. **可以同时为不同的用户做中继吗？**
   **完全可以，这就是标准的多租户（Multi-tenant）架构。** 你的 VPS 中转服务就像一个“盲眼邮递员”。它通过给每次传输分配独一无二的 Room ID（房间号），可以在内存中同时维护成千上万对用户的专属数据管道。A 用户传文件的流与 B 用户的流在物理和逻辑上绝对隔离。
 2. **VPS1 上需要安装什么服务？**
   **极简至上，千万不要装 MySQL 或 Redis。** 你只需要两样东西：
   * **反向代理（推荐 Caddy）**：负责接管 bob.bobbik.org 的流量，并全自动处理免费的 HTTPS 证书（现代浏览器要求 WebSocket 必须在 HTTPS 下运行）。
   * **Bob Relay Server (纯 Rust 单体程序)**：你用 Rust (tokio + axum) 编译出来的一个只有十几 MB 的轻量微服务。它不写硬盘，只在内存里做配对和字节流转发。
 3. **需要一个 index.html 吗？进度条怎么实现？**
   **极其需要！** 当你需要给没装 Bob Agent 的客户发文件时，客户点开链接看到的正是这个由你 VPS 托管的 index.html。
   * **进度条的真相**：VPS **不负责计算进度**。发送端（你的桌面 Agent）将大文件切成小块（Chunk）发送，网页端（或接收端 Agent）收到数据块后，前端 JS 简单计算 (已收字节 / 总文件大小) * 100%，并在本地丝滑地渲染出进度条和 MB/s 速度。
### 一、 用户需求梳理 (User Stories)
这套系统要解决泛白领群体在三种典型场景下的核心痛点：
 * **场景 1：内部联邦互联 (Agent to Agent)**
   * **痛点**：下班回家，想用家里的轻薄本调取公司台式机里的一个 500MB 项目文件。
   * **体验**：直接在家里的 Bob 聊天框输入：“提取公司电脑的 xxx 文件”。两端 Agent 后台通过 VPS 握手，跨越公司防火墙，满速将文件传回本地，Tauri 原生界面显示传输进度。
 * **场景 2：对外的极简分享 (Agent to Web / Web Drop) ??核心爆点**
   * **痛点**：微信发给客户的设计图有大小限制；用百度网盘客户嫌麻烦且限速，还容易泄露机密。
   * **体验**：你的 Agent 甩出一个阅后即焚链接（如 bob.bobbik.org/drop/x8a9）。客户在微信里点开，是一个极美的网页（index.html）。无需注册、无需下载 App，进度条走完，文件直接从你的电脑硬盘，穿过 VPS 管道，流进客户的手机里。
 * **场景 3：绝对隐私焦虑 (Privacy First)**
   * **需求**：商业数据绝不能在第三方服务器留存。必须实现端到端加密，确保哪怕 VPS 被黑客攻破，也只能看到乱码。
### 二、 技术架构需求 (Technical Specs)
为了实现以上体验，必须坚守两项硬核的工程铁律：
#### 1. 零落盘的中转枢纽 (Zero-Disk Memory Pipe)
VPS 端（Rust 服务）只做两件事：**握手信令交换** 和 **WebSocket 二进制流转发**。
当收到来自发送端的 File Chunk（如 1MB 一个的二进制块）时，VPS 在内存中瞬间将其 pipe 给接收端。**绝不在 VPS 的 SSD 上写入哪怕 1KB 的用户文件数据**。这既保护了隐私，又防止了服务器硬盘被撑爆。
#### 2. Web 端的完美端到端加密 (E2EE URL Hack)
既然经过 VPS，如何保证客户在 index.html 下载的文件不被中途窃听？
 * **黑科技**：Bob Agent 生成的链接形如 https://bob.bobbik.org/drop/x8a9#SecretKey123。
 * **原理**：在 HTTP 协议规范中，URL 里 # 后面的哈希值（Hash Fragment）**永远不会发送给服务器**。网页前端加载后，通过 JS 读取 URL 里的 SecretKey123。VPS 转发过来的是加密的乱码块，浏览器前端利用 Web Crypto API 在本地瞬间解密，再拼接拼成完整文件。VPS 全程是个“瞎子”。
### 三、 极简落地路径 (Implementation Roadmap)
建议将这个宏伟的网络工程拆分为三个“小步快跑”的里程碑（Milestone）：
#### ?? 阶段 1：点亮云端中继塔 (VPS 基础设施)
 1. 用 Rust (axum) 写一个不到 200 行代码的极简 WebSocket 服务。
 2. 功能只需一个：维持一个哈希表 HashMap<RoomID, Vec<WebSocket>>。当收到带 RoomID 的二进制包时，原封不动推给同房间的另一端。
 3. 编译后丢到 vps1 上，通过 Caddy 暴露 HTTPS。
#### ?? 阶段 2：跑通对外 Web Drop (MVP 优先实现)
*(最先做这个，它能给你极强的成就感和震撼的演示效果)*
 1. **静态页托管**：写一个极简的 Vue 3 单页 index.html（包含美观的进度条），直接让 axum 一并托管。前端逻辑：连接 WebSocket -> 接收乱码分片 -> 解密拼接 -> 渲染进度 -> 触发原生下载。
 2. **桌面端改造**：在 tools.rs 加一个新能力。当 Agent 接到指令，按 1MB 将目标文件切片并加密，连接 VPS 房间并不断泵送流数据。
#### ?? 阶段 3：打通对内联邦互传 (Agent Swarm)
 1. **设备身份**：给每个安装了 Bob Agent 的设备生成固定的公私钥对（Device ID）。
 2. **RPC 指令透传**：利用现有的中继通道，不仅能传文件，还能传 JSON。设备 A 的大模型可以输出一个 Tool Call 指令，通过 VPS 投递给设备 B，设备 B 在后台执行（例如 list_dir 浏览目录）后，将结果顺着网线传回给 A。
至此，你的 Bob Agent 就彻底超越了单机工具的范畴，成为了一张由你完全掌控的、去中心化的商业协同网络。

20260619_2nd Pass with Deepthink
这份“纯正零信任 P2P 直连”的路线图，剥离了所有沉重的云端包袱，将研发精力 100% 聚焦于**本地网络穿透、异步队列管理**与**大模型交互打通**上。
按照你的 Tauri (Rust + Vue 3) 架构底色，整个 P2P 联邦网络的开发可以拆解为四个“小步快跑”的敏捷冲刺阶段（建议作为一个月的 Milestone 纳入你的 todo.md）。
### ?? 阶段一：搭建“盲眼红娘”与联邦身份 (信令层)
这个阶段的重点是建立设备间的信任链，并让它们能在茫茫互联网中互相发现，且绝对不触碰真实数据。
 * **1. 极简信令服务器 (vps1 端)**
   * **开发任务**：使用 Rust (tokio + axum) 编写一个仅处理 WebSocket 连接的微服务。
   * **核心逻辑**：维护一个内存表。设备上线时发送 {"action": "register", "device_id": "xxx", "ip": "..."}。当设备请求建立连接时，服务器将双方的 IP 坐标作为“相亲名片”进行互换。
   * **部署**：编译为二进制文件，丢到 vps1 上，通过 Caddy 反向代理开启 WSS (WebSocket Secure)。
 * **2. 本地“联邦主密钥”生成 (bob-agent 端)**
   * **开发任务**：在 lib.rs 或 config.json 初始化时，生成一串助记词形式的 **Swarm Key**（如 apple-quantum-river）。
   * **UI 交互**：在前端 SettingsView.vue 新增【设备联邦】面板，显示本机的 Swarm Key。用户在另一台设备的向导中填入此 Key，即可完成“加群”。
   * **安全机制**：该 Key 作为 AES-GCM 的对称加密密钥，绝不上传到 VPS。
### ?? 阶段二：打通端到端直连与“异步待办” (传输层)
将物理限制（设备必须同时在线）通过优秀的工程设计转化为白领的“无感体验”。
 * **1. 植入 P2P 引擎 (UDP 打洞)**
   * **开发任务**：在 Tauri 架构中实现打洞逻辑。如果选择前端方案，在 tauri-bridge.js 中封装 RTCPeerConnection (WebRTC)；如果选择纯后端方案，在 src-tauri/Cargo.toml 中引入 iroh 库。
   * **核心逻辑**：接收到 VPS 传来的目标 IP 后，尝试 UDP 直连。一旦握手成功，建立端到端加密的 Data Channel。
 * **2. 构建“AI 待办队列” (SQLite 持久化)**
   * **开发任务**：在本地 bob.db 数据库中新增 pending_transfers 表（包含：目标设备、文件路径、重试次数、状态）。
   * **守护进程**：在 Rust 后端写一个定时轮询任务（如每 30 秒检查一次）。如果发现队列中有文件，且通过 VPS 信令查到目标设备刚刚上线，立刻在后台无声启动直连传输。
### ?? 阶段三：赋予大模型网络手脚 (Agentic 层)
让 Bob Agent 真正理解并掌握跨设备流转的能力，实现“上下文接力”。
 * **1. 扩展工具引擎 (tools.rs)**
   * **新增工具**：注册 send_to_device(device_name, file_path) 和 check_swarm_status() 供 LLM 调用。
   * **上下文打包逻辑**：当大模型执行跨端文件发送时，赋予它将当前的 conversation_id、相关 messages 以及选定的文件，打包成一个极小的 .zip（或自定义的二进制包）的能力。
 * **2. 接收端的“魔法唤醒”**
   * **开发任务**：当目标设备收到包裹后，Rust 后端解析包内容，将历史对话差异并入本地的 SQLite 数据库。
   * **UI 触达**：触发 app.emit("swarm:received")。Vue 前端弹出一个极其优雅的通知：“*收到来自 [办公电脑] 的 150MB 研报及上下文，是否继续上次的探讨？*”
### ?? 阶段四：打磨“傲骨式”隐私交互 (体验层)
将偶尔出现的网络失败，转化为用户建立绝对信任的绝佳契机。
 * **1. 防火墙降级弹窗**
   * **开发任务**：当 WebRTC 或 iroh 连续打洞失败（通常是因为极端对称型 NAT）时，拦截默认的系统报错。
   * **UI 交互**：弹出定制的盾牌图标警告卡片：*“因当前网络防火墙极其严苛，直连被阻断。为捍卫您的商业机密，Bob 拒绝使用云端中转。文件已放入待办队列，建议您切换网络环境后重试。”*
 * **2. 局域网“超光速”感知**
   * **UI 交互**：当两台设备处于同一 Wi-Fi，Agent 走内网直连时，在进度条旁边显示“?? 局域网加密直连中”的专属 Badge，让用户明确感知到当前传输没有浪费任何外网带宽。
