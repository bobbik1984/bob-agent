# Gemini Deep Research 提示词

> **使用方法**：以下五个提示词分别对应专家组讨论框架中的五个工作流。建议**逐个投递**（Deep Research 一次专注一个主题效果最好），而非一次性全部塞入。每个提示词已内置了必要的背景上下文摘要，无需额外附件。
>
> **投递时**：将下方某个提示词整段复制粘贴到 Gemini Deep Research 的输入框中即可。如果 Deep Research 支持附件，可以同时附上《分布式 Agent 认知系统审视.docx》原文以获得更好的理解。

## 提示词 1：场景发现与需求验证

```
我正在构建一个桌面端 AI 助手产品。它的独特之处在于：用户通过一个极简的桌面对话界面（类似 ChatGPT 客户端）下达复杂任务后，系统后台不是用单一大模型即时回答，而是启动一套多智能体认知演化引擎——多个功能解耦的 AI 角色（负责质疑假设的、负责强制发散的、负责压力测试可行性的）在一个共享的知识图谱"黑板"上进行数小时的自主对抗与迭代，最终只将经过层层淘汰的高置信度结论推送给用户。

核心产品命题是"人机交互时间的不对称性"：用户投入 1 分钟，系统后台深度演化数小时，交付专家级成果。

请帮我做一次深度研究，回答以下问题：

1. **真实需求验证**：在现实世界中，哪些职业群体或工作场景确实面临"问题复杂度超出单次 AI 对话能力边界"的困境？请列举至少 8 个具体的职业角色和工作场景（不要泛泛而谈"研究人员"，而是具体到如"对冲基金的宏观策略分析师在季度调仓前需要同时权衡 15+ 个相互矛盾的宏观指标"这种粒度）。

2. **"等待意愿"调研**：是否有用户研究、产品案例或学术文献表明，用户在某些条件下愿意主动选择"等待数小时获得深度结果"而非"即时获得浅层回答"？这种"异步深度思考"模式在哪些已有产品中被验证过（如 Elicit、Google Deep Research、法律 AI 工具等）？用户满意度和留存率如何？

3. **反场景分析**：哪些看似适合多智能体深度推演的场景，实际上单模型 + 好提示词就足够解决？请给出判断标准——什么特征的任务值得调用这种重型后台，什么特征的任务是"杀鸡用牛刀"？

请在报告中附上所有引用来源的链接。
```

## 提示词 2：认知黑板的工程化设计

```
我在设计一个"认知黑板"系统——本质上是一个多智能体共享的结构化状态存储。当多个 AI 智能体围绕一个复杂问题进行异步辩论时，它们的论点、假设、反驳和证据都作为节点和关系写入这个图谱式的黑板中（而不是简单追加到聊天记录）。

我面临几个核心的工程困境，请帮我做深度研究：

1. **信息分级与上下文污染防治**：
   在多轮辩论中，黑板上会积累大量的中间假设、已被否定的论点、低置信度猜想。后来参与辩论的 AI 容易将这些"废料"当作事实吸收（即"嵌套幻觉"问题）。
   - 请调研：在非 AI 领域（如版本控制系统 Git 的 branching/tagging 模型、学术论文的同行评审状态机、Wikipedia 的条目质量分级制度、法律体系中的证据等级制度），有哪些成熟的信息分级与生命周期管理方案可以借鉴？
   - 在 AI/多智能体领域，是否有论文或项目实现了黑板内容的自动分级（如 fact / hypothesis / refuted / pending_verification）？

2. **认知遗忘与记忆巩固**：
   人类认知的关键能力之一是"遗忘"——在睡眠中将短期记忆压缩为长期记忆，丢弃噪声。
   - 请调研：在 AI 系统中，是否有类似"记忆巩固"（Memory Consolidation）或"选择性遗忘"（Selective Forgetting）的机制被提出或实现过？特别关注 LLM 长期记忆管理、知识图谱的自动精简/聚合、以及 MemWalker / MemGPT 等项目。
   - 神经科学中关于睡眠记忆巩固的研究，是否有可以直接映射到 AI 系统设计的原则？

3. **跨模态黑板**：
   当黑板上同时存在文本、表格数据、图片分析结果、代码执行输出时，如何在同一语义空间中比较和关联它们？是否有关于"多模态知识图谱"的前沿工作？

请在报告中对每个问题给出：(a) 现有方案综述，(b) 各方案的优缺点对比，(c) 你的推荐路径。附上所有引用来源。
```

## 提示词 3：多智能体收敛机制的可行性评估

```
我正在为一个多智能体辩论系统设计"收敛刹车机制"——即如何客观判断"讨论已达最优解，可以停止"，而不依赖人类直觉。

我的理论框架提出了一个三标尺联合收敛条件：
- 标尺 1：语义熵（Semantic Entropy）—— 将多个智能体的输出映射到语义向量空间，计算聚类的分散度。高语义熵 = 分歧大；低语义熵 = 趋于共识。
- 标尺 2：信息增益（Information Gain）—— 每轮辩论引入的"新的可操作信息"相对于全局上下文的边际贡献。连续多轮趋近于零 = 讨论已触及能力天花板。
- 标尺 3：状态机异常检测 —— 监控工具调用历史和输出模式，识别循环论证和停滞螺旋。

请帮我做深度研究：

1. **语义熵的工程可行性**：
   - 语义熵的概念最早由哪些论文提出？目前有哪些具体的计算方法（clustering-based、sampling-based 等）？
   - 在一个典型的 5 轮多智能体辩论场景中，如果每轮需要对 3-5 个智能体的输出计算一次全局语义熵，额外的 API 调用成本和计算时间大约是多少？是否存在低成本的近似方法？
   - 有没有已有的开源工具或库可以直接使用？

2. **循环论证检测**：
   - 请深入调研 LLM 循环推理（circular reasoning）和智能体停滞螺旋（stagnation spirals）的最新研究。有哪些检测方法被提出（隐状态分析、工具调用模式匹配、输出相似度阈值等）？
   - 是否有论文对比过这些方法的检测准确率和误报率？

3. **更简单的替代方案**：
   - 如果不使用精确的语义熵计算，仅依靠朴素规则（如"连续 N 轮输出的 ROUGE-L 相似度 > 0.85 + 无新工具调用"），能否达到 80% 的收敛判断准确率？是否有相关的基准测试或经验数据？
   - 请调研工业界（如 AutoGen、CrewAI、LangGraph）实际采用的收敛/终止策略是什么——是精确的数学度量，还是简单的轮次上限 + 启发式规则？

请给出一份对比表格：精确方法 vs. 近似方法，从准确率、计算成本、实现复杂度三个维度进行比较，并给出你的推荐。
```

## 提示词 4：人机信任与交互设计模式

```
我正在设计一种新型 AI 产品交互模式：用户通过桌面助手下达复杂任务后，AI 不是即时回答，而是在后台独立"思考"数小时（多个 AI 角色自主辩论、验证、迭代），最终推送一份深度报告。

这种"异步深度思考"模式对用户心理模型提出了前所未有的挑战。请帮我做深度研究：

1. **透明度与信任的学术研究**：
   - 当 AI 系统从"即问即答的工具"变成"后台独立运行数小时的自主体"时，用户心理模型会发生什么变化？是否有人机交互（HCI）领域的研究专门探讨过这种"延迟交付型 AI"的信任构建问题？
   - "可解释 AI"（XAI）领域的研究中，关于"渐进式透明（Progressive Disclosure）"的最佳实践是什么？是否有研究表明，展示完整的推理过程反而会降低用户信任（因为暴露了不确定性）？

2. **现有产品中的"异步 AI"交互模式**：
   - 请调研以下产品的交互设计，分析它们如何处理"AI 长时间后台运行"的用户体验：
     * Google Deep Research（Gemini 的深度研究功能）
     * Elicit（学术研究 AI）
     * Devin / Codex（自主编程 AI）
     * GitHub Copilot Workspace
     * 法律 AI 工具（如 Harvey、CoCounsel）
   - 这些产品在"进度展示"、"中间结果预览"、"人工介入节点"、"失败沟通"方面分别采用了什么设计模式？有哪些值得借鉴、哪些被用户批评？

3. **"高价值否定结论"的沟通设计**：
   - 当 AI 经过数小时深度推演后，得出的结论是"这个问题在当前信息条件下无法给出可靠答案"，如何向用户传达这一结论而不让用户觉得浪费了时间？
   - 是否有用户研究表明，"知道自己不知道"这件事本身具有价值？在医疗诊断、法律咨询等高风险领域，专业人士如何向客户传达"不确定性"？

请以产品设计师可直接使用的格式输出：(a) 设计原则清单（不超过 10 条），(b) 交互模式参考案例对比表，(c) 需要避免的反模式（Anti-Patterns）。
```

## 提示词 5：竞品格局与战略定位

```
我正在评估一个产品方向的竞争格局：一个常驻桌面的 AI 助手，其后台是一套多智能体认知演化引擎（多个 AI 角色自主对抗、淘汰、收敛），而非单一模型。核心差异化是"用算力换确定性"——通过后台长时间的自主推演，交付比单次 AI 对话深刻得多的结果。

请帮我做一次全面的竞品深度研究：

1. **竞品矩阵**：请按以下四个象限分类，找出每个象限中最有代表性的 3-5 个产品/项目，并分析它们的核心能力、商业模式、融资阶段、用户规模：
   - **象限 A：桌面端 AI 助手**（Apple Intelligence、Microsoft Copilot Desktop、各种开源 Tauri/Electron AI 壳如 Cherry Studio、Jan.ai、LM Studio 等）
   - **象限 B：多智能体框架/平台**（AutoGen、CrewAI、LangGraph、MetaGPT、Camel-AI、OpenHands、Mastra 等）
   - **象限 C：知识管理 + AI**（Notion AI、Obsidian + 社区插件、Mem.ai、Rewind/Limitless、个人知识图谱工具等）
   - **象限 D：深度研究/分析工具**（Elicit、Consensus、Perplexity Pro、Google Deep Research、各类法律/金融 AI 研究工具等）

2. **融合型竞品**：是否有产品/项目已经在尝试同时覆盖上述多个象限（即"桌面友好 UI + 多智能体后台 + 知识图谱持久化"的组合）？如果有，它们做到了什么程度？如果没有，为什么？

3. **护城河分析**：
   - 如果我的核心护城河定义为"用户专属知识图谱的长期数据沉淀"——请挑战这一假设。这种护城河在什么条件下会失效？
   - 大模型能力正在快速迭代。如果 12-18 个月后，单个模型（如 GPT-5 或 Gemini 3）的推理能力强大到不需要多智能体对抗就能产出同等质量的结果，这个方向的价值主张是否会被根本性瓦解？请给出正反两方的论证。

4. **时间窗口与进入策略**：
   - 这个方向是否存在一个"时间窗口"——即现在进入还有先发优势，但 2 年后就会被大厂的原生能力覆盖？
   - 对于一个 1-2 人的独立开发者团队，最务实的进入策略是什么？哪些组件应该自建，哪些应该依赖现有开源框架？

请输出一份包含定位矩阵图（2x2 或雷达图形式的文字描述）、每个竞品的一句话定位、以及"致命假设清单"（3-5 个支撑该方向的关键假设 + 每个假设被证伪时的应对策略）的完整报告。
```

## 使用建议

| 顺序 | 提示词 | 优先级 | 理由 |
|------|--------|--------|------|
| 1 | 提示词 5（竞品格局） | ⭐⭐⭐ | Deep Research 最擅长的就是广度搜索和对比分析，先摸清战场 |
| 2 | 提示词 1（场景验证） | ⭐⭐⭐ | 在投入技术设计之前，先验证需求是否真实存在 |
| 3 | 提示词 4（交互设计） | ⭐⭐ | 产品体验决定了技术再好用户也不用的风险 |
| 4 | 提示词 3（收敛机制） | ⭐⭐ | 技术核心难题，需要学术文献支撑 |
| 5 | 提示词 2（黑板设计） | ⭐ | 最偏工程实现，Deep Research 的优势相对较小，更适合自己动手实验 |
