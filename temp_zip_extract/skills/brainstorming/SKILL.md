# 头脑风暴技能 (Brainstorming Skill)

## 技能目标

帮助用户从多个不同角度分析问题，综合各方面优劣，得出更全面、理性的结论。通过标准化的多维度分析流程，让决策更具科学性和说服力。

---

## 工作流程

### 第一步：明确问题

在开始分析前，确保问题定义清晰：

- 用一句话重新陈述用户的核心问题
- 确认分析目标（决策型 / 优化型 / 探索型）
- 如果问题模糊，先向用户提问澄清，再进入分析

### 第二步：选择分析视角

根据问题类型，从以下模块中选择 **2~4 个最相关的视角** 进行深入分析。不要为了"全面"而把所有模块都堆上去——选择真正有洞察价值的。

| 模块 | 适用场景 |
|------|---------|
| **SWOT 分析** | 战略规划、竞争分析、个人/组织决策 |
| **PEST 分析** | 外部环境评估、市场进入、政策影响 |
| **成本效益分析** | 资源投入决策、项目立项、方案比较 |
| **风险分析** | 新项目启动、高不确定性决策 |
| **五力分析** | 竞争格局、行业进入壁垒评估 |
| **利益相关者分析** | 涉及多方协作、变革管理、谈判场景 |

> 详细的每个模块执行指南见 `references/analysis-modules.md`

### 第三步：执行分析

对每个选定的模块，输出结构化分析结果。格式要求：

- 每个维度用 **简洁的要点** 列出（3~5 条，不堆砌）
- 每条要点附带 **1 句具体说明**，避免空话
- 区分"已知事实"和"推断假设"

### 第四步：综合评估

汇总各视角的分析，进行横向比较：

1. **核心矛盾**：找出各视角中反复出现的关键张力点
2. **权重判断**：哪些因素对当前目标影响最大？说明理由
3. **方案对比**（如有多个选项）：用简明表格对比各选项的得分

### 第五步：决策建议

给出明确的建议，结构如下：

```
【推荐方向】一句话结论

【核心理由】2~3 条最有力的支撑论点

【主要风险】需要特别关注的 1~2 个风险点

【下一步行动】具体可执行的 1~3 个步骤
```

---

## 输出规范

- **语言**：跟随用户输入语言（中文输入→中文输出）
- **长度**：分析深度与问题复杂度匹配，避免过度冗长
- **格式**：善用表格、要点列表，保持清晰的视觉层次
- **语气**：客观分析为主，建议部分可适度表达倾向性

## 快速参考

**触发此技能的典型问题类型：**
- "我应该选 A 还是 B？"
- "这个方案有哪些风险？"
- "帮我分析一下这个项目值不值得做"
- "如何提高 X？"（需要多方案对比时）

**不适合此技能的场景：**
- 单纯的事实查询（用搜索代替）
- 纯技术实现问题（用代码/工具代替）
- 创意发散（不需要结构化评估时）
