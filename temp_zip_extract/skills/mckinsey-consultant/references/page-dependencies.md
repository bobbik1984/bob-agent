# Dummy Page 依赖关系与前置条件规范

## 依赖类型定义

| 符号 | 类型 | 说明 | 示例 |
|:-----|:-----|:-----|:-----|
| ✅ 独立 | INDEPENDENT | 无依赖，可直接生成 | 封面、普通数据分析页 |
| ⏩ 依赖前页 | FORWARD | 需要前面特定页面的数据 | 趋势对比页、深入分析页 |
| ⏪ 依赖后页 | BACKWARD | 需要后面页面完成后总结 | 目录、执行摘要、章节总结 |
| 📄 需要文档 | REQUIRES_DOC | 需要用户提供特定文档 | 执行摘要(需 Issue Tree) |
| ⏸️ 最后生成 | BIDIRECTIONAL | 前后都依赖 | 目录、全局总结 |

---

## 标注格式

每个 Dummy Page 必须包含以下字段：

```markdown
## 第X页: [McKinsey 论点标题]

**依赖关系**: [符号 + 类型]
**前置条件**: [具体要求]
**缺失时对策**: [如果条件不满足的应对方案]

**布局**: [布局类型]
**图表**: [图表类型]
**数据需求**: [X轴/Y轴/系列]
**信息来源**: [web_search 关键词]
```

---

## 建议生成顺序

在 Dummy.md 开头必须添加"页面依赖关系总览"，按三轮顺序组织：

```markdown
## 页面依赖关系总览

**第一轮：独立页面** (可任意顺序)
- 第1页 (封面) ✅ 独立
- 第3-15页 (数据分析页) ✅ 独立

**第二轮：前向依赖页面** (依赖第一轮)
- 第16页 (章节总结) ⏩ 需要第3-15页
- 第18页 (深入分析) ⏩ 需要第5,7页

**第三轮：后向依赖页面** (最后生成)
- 第2页 (执行摘要) ⏪ 需要第3-24页
- 第25页 (结论建议) ⏸️ 最后生成
```

## 各类型示例 (各一个)

### 独立页面

```markdown
## 第5页: 中国新能源汽车市场规模持续扩大，2023年达1.2万亿元

**依赖关系**: ✅ 独立
**前置条件**: 无
**缺失时对策**: 无需

**布局**: 标题+单图表型
**图表**: 柱状图 (2019-2023 市场规模)
**信息来源**: web_search "中国新能源汽车市场规模 2023"
```

### 前向依赖

```markdown
## 第8页: 特斯拉市占率从25%降至18%，本土品牌崛起

**依赖关系**: ⏩ 依赖前页
**前置条件**: 需要第5页的"市场规模数据"作为分母
**缺失时对策**:
  1. 先完成第5页 (推荐)
  2. 临时搜索市场规模数据，直接生成
  3. 跳过，稍后再生成

**布局**: 标题+左右分栏型
**图表**: 左饼图(市占率) + 右柱状图(销量变化)
```

### 后向依赖

```markdown
## 第2页: 执行摘要

**依赖关系**: ⏪ 依赖后页
**前置条件**: 所有正文页(第3-20页)完成
**缺失时对策**:
  1. 推荐最后生成此页
  2. 如用户急需 → 基于 Issue Tree 生成框架，标注"[待完善]"
  3. 如用户提供 Hypothesis Tree → 基于假设生成执行摘要框架
```

## Agent 生成前检查规则

生成任何页面前，Agent 必须：

1. **读取该页的依赖关系标注**
2. **如果是 ✅ 独立** → 直接生成
3. **如果有依赖** → 检查依赖页面是否已完成
   - 已满足 → 从依赖页获取数据，继续生成
   - 未满足 → **告知用户**并提供选项（先完成依赖页 / 临时搜索 / 跳过）
   - **等待用户确认后再继续**
4. **如果是 ⏪ 后向依赖或 ⏸️ 最后生成** → 建议最后生成

---

## 跨对话续写要点

用户上传 Dummy.md 后，Agent 必须：

1. 先读取"页面依赖关系总览"
2. 检查用户要生成的页面类型
3. 自动执行依赖检查
4. 如有缺失，主动询问而非假设
5. 提供灵活应对策略

---

## 实施要点

### STEP 5 设计 Dummy 时

- 为每一页标注依赖关系
- 明确列出前置条件和缺失时对策
- 在 Dummy 开头添加"页面依赖关系总览"和建议生成顺序

### STEP 6-7 生成时

- 生成前自动检查依赖关系
- 如有缺失，提供清晰选项
- 等待用户确认后再继续
- 记录依赖数据的来源页面
